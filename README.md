# Furniture Management System

## Overview
A backend-focused Furniture Management System developed using Python, Django, and SQLite. The project is designed to manage furniture listings, user roles, and transaction records through a structured relational database architecture.

## Tech Stack
* Python
* Django
* SQLite
* Django Admin

### Key Features
#Role-Based Access Control
* Seller Management
* Buyer Management
* Rental User Management
* Dedicated profile and contact information for each user type

### Furniture Categorization
* New Furniture
* Old Furniture
* Rental Furniture

### Product Management
* Add furniture listings
* Update furniture details
* Delete furniture records
* Track furniture categories

### Database Design
* Structured relational database models
* Foreign key relationships
* Data integrity enforcement
* Scalable backend architecture

### Transaction Management
* Record user transactions
* Track furniture ownership and rentals
* Maintain transaction history

###Admin Panel
* User management
* Furniture management
* Transaction monitoring
* Centralized administrative control

## Project Highlights
* Designed normalized database models using SQLite.
* Implemented strong model relationships between users, furniture items, and transactions.
* Developed scalable backend logic following Django best practices.
* Ensured data consistency and integrity across all modules.
* Utilized Django Admin for efficient data management and monitoring.
